Chapter 6

The box model – forms a box around text to separate it from other content on the web page

  • Content - The text and web page elements in the container
  • Padding - The area between content and border
  • The Padding Property - The padding-top, padding-right, padding-left, padding-bottom
  • Configures the space between the html element and the border
  • For example:
    h1 {padding: 0;} h1 {padding: 20px 10px;} h1 {padding: 20px 30px 0 30px;}
  • Border – area between the padding and the margin
  • Margin – determines the empty space between the element and adjacent elements
  • The Marin Property – margin-top, margin-right, margin-left, margin-bottom
  • This configures the space between the element and adjacent elements
  • For example:
    h1 {margin: 0;} h1 {margin: 20px 10px} h1 {margin: 20px 30px 0 30px;}

The Box Sizing Property

  • Used to direct the browser to calculate the width and height of an element to include the value for content, padding, and border
  • You can use the universal selector (*) to apply this to all elements of the webpage
  • * {box-sizing: border-box;}

Normal flow – when the browser displays the elements in the order they were coded in

  • Float Property
    1. You can make an image appear to float on the right or left side of the browser window
    2. .pull-right{float:right} - in the style section

    3. - in the paragraph section

  • Clear Property
    1. Clears of terminates a float meaning the next text or section will be displayed in the normal flow
    2. Clear: left, right, or both
    3. <p class="bottom">
  • Overflow – configures the display of elements on a web page
    1. Useful to clear or terminate a float before the end of a container element
    2. Overflow: auto, hidden, scroll

    CSS Display Property – configure how and if an element is displayed

    • Display: none – element is not displayed
    • Display: block – element is displayed as a block element, even if it is actually an inline element
    • Display: inline – element will be rendered as an inline element even if it is actually a block element
    • Display: inline-block – element will display as an inline display element but can also be configured with properties of block display elements including height and width

    Vertical Navigation

  • <nav>

    <ul>

    <li> <a href="index.html">Home</a></li>

    <li><a href="menu.html">Menu</a></li>

    <li><a href="directions.html">Directions</a></li>

    </ul>

    </nav>

  • To remove the list marker and underline
    1. Nav ul {list-style-type: none;}
    2. Nav a {text-decoration: none;}

Horizontal Navigation

  • <nav>
  • <ul>
  • <li> <a href="index.html">Home</a></li>
  • <li><a href="menu.html">Menu</a></li>
  • <li><a href="directions.html">Directions</a></li>
  • </ul>
  • </nav>
  • To remove the list marker, underline, add padding, and configure inline display
  • Nav ul {list-style-type: none;}
  • Nav a {text-decoration: none; Padding-right: 10px;}
  • Nav li {display: inline;}

The Anchor Element

  • Link – default state for a hyperlink
  • Visited – a hyperlink that has been visited
  • Focus – triggered when the hyperlink has focus
  • Hover – triggered when the mouse moves over the hyperlink
  • Active – triggered when the hyperlink has been clicked
  • a: link {color: #000066}
  • a: visited {color: #003366}
  • a: focus {color: #FF0000}
  • a: hover {text-decoration: none; color: #0099CC}
  • a: active {color: #FF0000}

    • Position Property

    • Static – default value where the element is rendered in the normal flow
    • Fixed – configures the location of an element within the browser viewport; element doesn’t move when the page is scrolled
    • Nav {position: fixed}
    • Relative – configures the location of an element relative to where it would otherwise render in normal flow
    • P {position: relative; Left: 30px; Font-family: arial, sans-serif;}
    • Absolute – configures the location of an element outside of normal flow precisely in its container element
    • P {position: abolute; Left: 200px; Top: 100px; Font-family: arial, sans-serif; Width: 300px;}

    Header Element

    • Block display – contains the headings of either a web page document or an area in the document such as the section or article

    Nav element

    • Block display – contains a section of hyperlinks

    Main element

    • Block display – contains main page content

    Footer element

    • Block display – contains the footer content of a web page or specific area such as a section or article on a web page

    Aside element

    • Block display – contains a side bar, note or other tangential content

    Section element

    • Block display – contains a section of a document such as a chapter or topic

    Article element

    • Block display – contains an independent entry such as a blog posting, comment, or e-zine article that could stand on its own

    Time element

    • Inline display – represents a date or time, could be useful to date articles or blog posts

    To configure a class

    • If the style may apply to more than element on a page
    • Use the .(dot) notation in the style sheet
    • Use the class attribute in html

    To configure an id

    • If the style is specific to only one element on a page
    • Use the # notation in the style sheet
    • Use the id attribute in the html

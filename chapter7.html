Chapter 7

Fragment identifiers

  • A link to part of a web page
  • Also called fragments
  • Has two components
    1. The element that identifies the named fragment of a web page – requires the id attribute <div id="top"> …. </div>
    2. The anchor tag that links to the named fragment of a web page – uses a href element <a href="#top"> Back to Top </a>
  • Opening a link in a new browser window
  • The target attribute on the anchor element opens a link in a new browser window or a new tab <a href="http://yahoo.com" target="_blank"> Yahoo!</a>

HTML block anchor

  • Configure block display elements within a hyperlink

    <a href=http://www.w3.org>

    <h1> HTML 5 Reference </h1>

    <p> book mark this site for a handy HTML reference </p> </a>


Telephone or text hyperlinks

  • Telephone scheme – Many mobile phones will initiate a phone call when the hyperlink is clicked

    <a href=tel:888-555-5555>Call 888-555-5555</a>

  • SMS scheme – Many mobiles will initiate a text when the hyperlink is clicked

    <a href="sms:888-555-5555"> Text 888-555-5555</a>

Sprite

  • An image file that contains multiple small graphics
  • Saves download time
  • The first image appears at the top and the second image appears 100 pixels down from the top

Three Column Page Layout

  • Has a header across the top with three columns below including: left, right, and center column
  • Left-column navigation – Float: left; width:150px;
  • Right-column navigation – Float: right; width:200px;
  • Center column – Uses the remaining screen available { Margin: 0 210px 0 160px; }
  • Footer – Clears the float {Clear: both;}

CSS Print Styling

  • Create an external style sheet with the configurations for browser display
  • Then create an external style sheet with the configurations for printing
  • Connect both external style sheets to the web page using two <link> elements

    <link rel="stylesheet" href="wildflower.css" type="text/css" media="screen">

    <link rel="stylesheet" href="wildflowers.css" type="text/css" media="print">

Print Styling Best Practices

  • Hide non-essential content – #nav {display: none}
  • Configure font size and color for printing – Use pt font sizes, use dark font color
  • Control break pages – .newpage {page-break-before: always}
  • Print URLs for hyperlinks – #sidebar a:after {content: "("attr(href)")";}

Mobile Web Design Best Practices

  • Optimize layout, navigation, graphics, and text for mobile use
  • Design for one web
    1. Use a single column design
    2. Scroll only one way
    3. Use heading elements
    4. Use lists
    5. Avoid using tablets
    6. Provide labels for form controls
    7. Avoid using pixel units in style sheets
    8. Hide content that is not essential for mobile use
    9. Provide minimal navigation near the top of the page
    10. Provide consistent navigation
    11. Avoid hyperlinks that open files in new windows or pop-up windows
    12. Try to balance both the number of hyperlinks on a page and the number of levels needed to access info
    13. Avoid displaying images that are wider than the screen width
    14. Configure alternate, small optimized background images
    15. Some mobile browsers will downsize all images so avoid using images that contain text
    16. Avoid the use of large graphic images
    17. Specify the size of images
    18. Provide alternate text for graphics and other non-text elements
    19. Configure good contrast between text and background colors
    20. Use common font typefaces
    21. Configure font sizes with em units or percentages
    22. Use a short, descriptive page title

Viewport Meta Tag

  • Configures width and initial scale of browser viewport – <meta name="viewport" content="width"device-width, initial-scale=1.0">
  • Media query – Determines the capability of the mobile device such as screen resolution and directs the browser to styles configured specifically for those capabilities

    • <link href="lighthousemobile.css" rel="stylesheet" Media="only all and (max-device-width:480px)">

  • Within CSS - media only all and (max-width: 768px){ }

Flexible images – to make images display on different screens

  • Edit HTML – remove height and width attributes
  • CSS – img {max-width: 100%; height: auto;

Responsive Images

  • <picture>

    <source media="(min-width: 800px)" srcset="medium.jpg">

    <source media="(min-width: 320px)" srcset="small.jpg">

    <img src="fallback.jpg" alt="waterwheel">

    </picture>
